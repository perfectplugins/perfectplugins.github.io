PerfectPortals

12 stunning colored portal pads with warp system, server transfers, and a built-in reverse proxy for multi-server networks. Link portals, teleport players, or route them between servers — all from one plugin.

Version2.0
PlatformHytale Server Plugin
Java21
DependenciesNone

Installation

  1. Download PerfectPortals-2.0.jar
  2. Place it in your server's UserData/Mods/ folder
  3. Start the server — config and data files auto-generate
  4. Give yourself a portal: /give Portal_Red

Warp System

Link any two portals together for instant bidirectional teleportation.

  1. Place two portals anywhere in the world
  2. Open /pportals — the admin UI shows all placed portals
  3. Click portal A in the list, then portal B
  4. Click "Link A - B"
  5. Walk up to either portal and press [F] to teleport

Links are bidirectional — both portals teleport to each other. Breaking a linked portal auto-cleans the link.

Server Transfer

Portals can transfer players to another Hytale server. Configure via the admin UI:

  1. Open /pportals, select a portal
  2. Click "Server Config"
  3. Set Host, Port, Countdown, and Label
  4. Players pressing [F] on that portal get transferred to the target server

Proxy Setup (v2.0)

The built-in reverse proxy hides backend server IPs. Players connect to one address; the proxy routes them.

Players ──► proxy:5520 ──► Lobby (localhost:5521)
           proxy:5530 ──► SMP   (172.18.0.3:5520)
           proxy:5540 ──► PvP   (172.18.0.4:5520)

Step 1: Change Lobby Port

Set your Hytale lobby server to listen on a different port (e.g. 5521).

Step 2: Configure proxy.json

File: UserData/Saves/MODDING/mods/KatsuyaTV_PerfectPortals/proxy.json

{
  "enabled": true,
  "publicHost": "play.yourserver.com",
  "backends": {
    "lobby": { "listenPort": 5520, "target": "localhost:5521" },
    "smp":   { "listenPort": 5530, "target": "172.18.0.3:5520" },
    "pvp":   { "listenPort": 5540, "target": "172.18.0.4:5520" }
  }
}

Step 3: Set Portal Labels

In the portal Server Config UI, set the Label to match the backend name (e.g. smp). The label is case-sensitive.

Step 4: Open Firewall Ports

Open all proxy listen ports (5520, 5530, 5540, etc.). Backend internal ports do NOT need to be exposed.

FAQ

Can players connect directly to backend servers?

Not if you configure your firewall to block external access to backend ports. Only the proxy ports should be public.

Do backend servers need PerfectPortals?

No. Only the lobby server needs the plugin. Backends are regular Hytale servers.

Can I use portals for same-server teleportation AND server transfer?

Yes. Portals without a server config act as local warps. Portals with server config transfer to other servers.
